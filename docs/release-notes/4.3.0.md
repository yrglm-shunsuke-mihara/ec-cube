# EC-CUBE 4.2.3から4.3.0 技術者向けリリースノート

## 概要
EC-CUBE 4.3.0は、コミュニティの力を結集したメジャーアップデートです。Symfony 6.4への大幅なフレームワーク更新、セキュリティ強化、パフォーマンス改善、そして多数の新機能が含まれています。

## 🔧 主要な技術変更

### フレームワーク・依存関係の大幅更新

**Symfony 6.4への更新**
- 全Symfonyコンポーネントを6.4系に統一更新
- `symfony/framework-bundle: ^6.4`
- `symfony/security-bundle: ^6.4`
- `symfony/form: ^6.4`
- `symfony/twig-bundle: ^6.4`
- その他20以上のSymfonyコンポーネント更新

**PHP要件**
- PHP 8.1.x / 8.2.x / 8.3.x サポート
- PHP 8.1以上が必須要件

**Doctrine ORM更新**
- `doctrine/orm: ^2.11`
- `doctrine/dbal: ^3.3`
- `doctrine/doctrine-bundle: ^2.0`
- `doctrine/doctrine-migrations-bundle: ^3.1`

### セキュリティ強化

**パスワードハッシュ機能の刷新**
```yaml
# 旧: security.encoders
security:
    password_hashers:
        legacy:
            id: 'Eccube\Security\PasswordHasher\PasswordHasher'
        Eccube\Entity\Customer:
            algorithm: 'auto'
            migrate_from:
                - legacy
        Eccube\Entity\Member:
            algorithm: 'auto'
            migrate_from:
                - legacy
```

**セッション管理の改善**
- 新しい`Eccube\Session\Session`クラスの導入
- `RequestStack`ベースのセッション管理に変更
- セッションハンドリングの安全性向上

**認証システムの更新**
- `UserPasswordHasherInterface`への移行
- `EncoderFactoryInterface`から`PasswordHasherInterface`への変更
- より安全なパスワード検証機能

### アーキテクチャ改善

**コントローラー基盤の強化**
```php
// AbstractController.php の主要変更
- use Symfony\Component\HttpFoundation\Session\SessionInterface;
+ use Eccube\Session\Session;
+ use Symfony\Component\Routing\RouterInterface;

protected $router; // 新規追加
public function setSession(Session $session) // 型変更
public function setRouter(RouterInterface $router) // 新規追加
```

**依存性注入の改善**
- RouterInterfaceの直接注入
- セッション管理の統一化
- サービスコンテナの最適化

## ⚠️ 破壊的変更・注意事項

### 必須対応事項

1. **セッション関連**
   - `SessionInterface` → `Eccube\Session\Session`への変更
   - カスタムコントローラーでセッション使用時は型変更が必要

2. **セキュリティ関連**
   - パスワードエンコーダーからハッシャーへの変更
   - 認証関連のカスタマイズがある場合は見直しが必要

3. **Symfony 6.4対応**
   - 非推奨機能の削除
   - 設定ファイルの構文変更

### 推奨対応事項

1. **プラグイン更新**
   - 既存プラグインの4.3対応確認
   - 依存関係の更新

2. **カスタマイズ見直し**
   - コアファイルのオーバーライドがある場合の確認
   - テンプレートファイルの互換性確認
